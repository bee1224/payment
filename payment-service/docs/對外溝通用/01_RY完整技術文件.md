# RY 完整技術文件

> 最後整理：2026-07-09
> 這份文件保留 RY 歷史規格的整理用途，但 `payment-service` 目前實際支援範圍已縮減，請先看下方「目前服務實際支援範圍」。

## 目前服務實際支援範圍

### 代收 API

- `POST /api/pay_order`
- `POST /api/query_transaction`

### 目前代收渠道

| `pay_channel_id` | `channel_code` | 說明 |
|---|---|---|
| `1000` | `CREDIT` | 信用卡一次付清 |
| `1001` | `APPLEPAY` | Apple Pay |
| `1002` | `GOOGLEPAY` | Google Pay |
| `1005` | `WEBATM` | WebATM |
| `1006` | `VACC` | ATM 虛擬帳號 |
| `1007` | `CVS` | 超商代碼 |
| `1008` | `BARCODE` | 超商條碼 |

### 已不再支援或不再輸出的內容

- 舊渠道 `1003`、`1004`
- 支付寶相關欄位與說明
- `alipay_qrcode`
- `qr_url`
- `expired`
- `rate`

### 目前回應特性

- `pay_amount` 以整數 TWD 金額處理。
- `view_url` 會導向 `/api/v1/deposits/{order_no}/redirect`。
- `real_price` 目前與 `bill_price` 相同。
- 查單只使用 `0`、`2`、`5` 三種狀態。
