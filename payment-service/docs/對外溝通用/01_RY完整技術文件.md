# RIG001 Gateway 技術總覽

> 文件版本：2026-07-11  
> 適用對象：合作商戶、技術對接窗口、專案協調人員

本文件用於說明 `RIG001 Gateway` 目前對外提供之技術能力、支援範圍與接口定位，供合作方於正式對接前快速掌握整體架構。若本文件與個別 API 文件描述不同，請以各 API 對接文件內容為準。

## 快速閱讀

如合作方技術人員需先快速掌握對接範圍，建議優先確認以下資訊：

| 項目 | 內容 |
|---|---|
| 正式對外域名 | `https://api.nnviopp.com` |
| 代收正式接口 | `POST /api/pay_order`、`POST /api/query_transaction` |
| 代付正式接口 | `POST /api/payouts`、`POST /api/payouts/query`、`GET /api/payouts/{payout_no}` |
| 代付回調 | 商戶建單時提供 `callback_url`，系統於終態時主動通知 |
| 銀行編碼 | 請參照 `04_銀行編碼.md` |

如需詳細欄位、範例請求與回調格式，請直接參閱個別 API 文件。

## 1. 服務定位

`RIG001 Gateway` 目前提供以下兩類能力：

- 代收服務
- 代付服務

正式對外域名如下：

- `https://api.nnviopp.com`

## 2. 目前支援範圍

### 2.1 代收接口

- `POST /api/pay_order`
- `POST /api/query_transaction`

### 2.2 代付接口

- `POST /api/payouts`
- `POST /api/payouts/query`
- `GET /api/payouts/{payout_no}`

### 2.3 代付結果通知

- 商戶如於建單時提供 `callback_url`，系統將於訂單進入終態後主動回調商戶系統。

### 2.4 代付餘額查詢

- `POST /api/payments/balance`

說明：
此接口目前保留供特定對接情境使用。如合作方需接入，請於對接前先與我方確認使用方式與權限配置。

## 3. 代收通道支援清單

目前支援以下 7 種代收方式：

| `pay_channel_id` | `channel_code` | 說明 |
|---|---|---|
| `1000` | `CREDIT` | 信用卡一次付清 |
| `1001` | `APPLEPAY` | Apple Pay |
| `1002` | `GOOGLEPAY` | Google Pay |
| `1005` | `WEBATM` | WebATM |
| `1006` | `VACC` | ATM 虛擬帳號 |
| `1007` | `CVS` | 超商代碼 |
| `1008` | `BARCODE` | 超商條碼 |

## 4. 重要整合說明

### 4.1 代收金額格式

- `pay_amount` 以新台幣整數金額處理。
- 回傳欄位中的 `bill_price` 與 `real_price` 目前為相同金額。

### 4.2 代收付款導頁

- 建單成功後會回傳 `view_url`。
- 商戶應引導使用者前往該網址完成付款流程。

### 4.3 代收查單狀態

目前查單接口僅使用以下狀態值：

| `status` | 說明 |
|---|---|
| `0` | 待付款 / 處理中 |
| `2` | 已付款 |
| `5` | 付款失敗 |

### 4.4 代付銀行編碼

- 代付接口中的 `pay_bank_name` 需傳入三碼銀行代碼。
- 銀行代碼請參照 `04_銀行編碼.md`。
