# 藍新代收串接說明

## MPG 送單

```mermaid
flowchart LR
  A[本地訂單] --> B[TradeInfo]
  B --> C[AES-256-CBC]
  C --> D["TradeSha: HashKey + 密文 + HashIV"]
  D --> E[POST MPG]
```

## MPG 欄位

| 欄位 | 來源 |
|---|---|
| `MerchantID` | `config.yaml` |
| `TradeInfo` | 加密後交易資料 |
| `TradeSha` | SHA256 驗證碼 |
| `EncryptType` | 固定 `0`，明確指定 AES/CBC/PKCS7 |

> 正式環境實測：首次 Notify 可能沒有 PKCS7 padding。驗簽成功後，接收端須相容有效的無填充 CBC 明文。
| `Version` | `2.3` |

## 通知處理

```mermaid
flowchart TD
  A[收到 notify] --> B{TradeSha 正確?}
  B -- 否 --> X[拒絕]
  B -- 是 --> C[解密 TradeInfo]
  C --> D{金額相符?}
  D -- 否 --> X
  D -- 是 --> E[更新訂單]
  E --> F{付款成功?}
  F -- 是 --> G[寫入代收帳本]
  F -- 否 --> H[標記失敗]
```

## 程式位置

| 檔案 | 用途 |
|---|---|
| `client.go` | 藍新 client |
| `crypto.go` | 加解密與驗簽 |
| `mapper.go` | 訂單轉 MPG 欄位 |
| `notify.go` | 通知資料解析 |
