# 對接規格確認清單

> 最後更新：2026-07-11

## 商戶側要確認

- 商戶是否明確知道正式對接對象是 `RIG001 Gateway`
- 商戶是否知道正式商戶號是 `RIG001`
- 商戶是否知道三方密鑰是我方發放 API Key，而不是藍新 `HashKey/HashIV`
- 商戶是否知道收款 callback 需回 `OK`
- 商戶是否知道代付正式 workflow 走 `/api/payouts/*`

## 我方內部要確認

- `MERCHANT_CODE` 是否要以 `RIG001` bootstrap
- `MERCHANT_API_KEY` 是否為目前要生效的正式 key
- `GATEWAY_*` 環境變數是否齊全
- VPS `.env` 是否為 Namecheap 新環境版本
- 是否已決定正式部署目錄 `/opt/payment-service`

## 上線前最後確認

- `go test ./...` 通過
- `/health` 可用
- 收款建單可用
- 代付建單可用
- 審核 token 可用
- callback 可打回商戶
