# RY 對接規格確認清單

> 更新日期：2026-07-09
> 這份清單以目前已收斂後的代收 / 代付範圍為準。

## 已確認

| 項目 | 結論 |
|---|---|
| 收款建立 API | `POST /api/pay_order` |
| 收款查單 API | `POST /api/query_transaction` |
| 代付建立 API | `POST /api/payments/pay_order` |
| 代付查單 API | `POST /api/payments/query_transaction` |
| 代付餘額 API | `POST /api/payments/balance` |
| 收款渠道 | 僅保留 `1000`、`1001`、`1002`、`1005`、`1006`、`1007`、`1008` |
| 收款 callback 狀態 | `30000` 成功、`50000` 失敗、`10000` 處理中 |
| 收款查單狀態 | `0`、`2`、`5` |

## 仍需確認

| 項目 | 現況 | 待確認內容 |
|---|---|---|
| 收款欄位建議值 | 補充欄位仍可帶入 | 各渠道對 `bank_account`、`store_number`、`bank_id` 的商務規範 |
| 代付銀行白名單維護流程 | 程式已有 whitelist 驗證 | 後續由誰維護與更新 |
| callback 重試機制 | 代收尚未正式 outbox 化 | 是否比照代付補齊 |

## 不再視為缺口

以下項目因目前產品範圍已收斂，不再列為規格缺口：

- `expired`、`qr_url`、`alipay_qrcode`、`rate` 是否依渠道出現
- 支付寶或其他已移除渠道的欄位矩陣
