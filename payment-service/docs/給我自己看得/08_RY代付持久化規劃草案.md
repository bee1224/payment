# 代付持久化現況與後續草案

> 最後更新：2026-07-11
> 檔名保留 `RY` 為歷史命名

## 現況

本系統代付 workflow 已不再只是單純代理。

目前已落地的核心資料包括：

- `payout_orders`
- `payout_beneficiaries`
- `payout_transactions`
- `payout_callbacks`
- `merchant_payout_callback_tasks`
- `merchant_balances`
- `ledger_entries`

## 本文件用途

本文件不再描述「是否要做持久化」，而是保留成：

- 已實作現況摘要
- 後續優化方向草案

## 已實作能力

- payout workflow 訂單持久化
- 多次 submission attempt 紀錄
- provider callback 入庫
- merchant callback task / retry / resend
- ledger 與 merchant balance 異動

## 後續可優化

- 將 callback retry 參數、退避策略與告警規則正式化
- 補更完整的人工審核 / 取消 / 退回狀態圖
- 若未來上游不只一個 provider，再抽象化 payout provider registry

## 目前部署前解讀

如果你是為了上板前判斷系統成熟度，結論是：

- 代付 workflow 已可部署
- 現在的主要風險已不是「沒有持久化」
- 主要風險轉為：文件命名殘留、正式部署 runbook、商戶 key 管理與實際上線驗證
