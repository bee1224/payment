# RY 收款串接規格

> 更新日期：2026-07-09
> 這份文件以目前程式實作為準，僅描述現在仍開放的代收渠道與回應欄位。

## 目前提供的 API

- `POST /api/pay_order`
- `POST /api/query_transaction`

相容路由 `POST /api/v1/deposits`、`POST /api/v1/deposits/query` 仍可用，但已標記為 deprecated。

## 目前支援的收款渠道

| `pay_channel_id` | 內部 `channel_code` | 說明 |
|---|---|---|
| `1000` | `CREDIT` | 信用卡一次付清 |
| `1001` | `APPLEPAY` | Apple Pay |
| `1002` | `GOOGLEPAY` | Google Pay |
| `1005` | `WEBATM` | WebATM |
| `1006` | `VACC` | ATM 虛擬帳號 |
| `1007` | `CVS` | 超商代碼 |
| `1008` | `BARCODE` | 超商條碼 |

## 建立收款訂單

### 請求欄位

| 欄位 | 型別 | 必填 | 說明 |
|---|---|---|---|
| `pay_customer_id` | string | Y | 商戶 ID |
| `pay_apply_date` | string | Y | Unix timestamp |
| `pay_order_id` | string | Y | 商戶訂單號 |
| `pay_notify_url` | string | Y | 商戶 callback URL |
| `pay_amount` | string / number | Y | 正整數 TWD 金額 |
| `pay_channel_id` | string | Y | 收款渠道編碼 |
| `bank_account` | array[string] | N | 若商戶需要自行指定收款帳號可帶入 |
| `store_number` | array[string] | N | 超商相關資訊，可選填 |
| `pay_product_name` | string | N | 商品描述 |
| `user_name` | string | N | 付款人姓名 |
| `bank_id` | string | N | 銀行代碼或補充資訊 |
| `pay_currency` | string | N | 補充資訊，會進 `remark` |
| `mobile` | string | N | 補充資訊，會進 `remark` |
| `id_no` | string | N | 補充資訊 |
| `pay_md5_sign` | string | Y | MD5 簽名 |

### 實作備註

- `pay_amount` 目前只接受整數 TWD 金額。
- `pay_channel_id` 會先映射成內部 `channel_code`，再交給 `newebpay` provider。
- `view_url` 固定導向 `/api/v1/deposits/{order_no}/redirect`，由服務端產生自動提交到 NewebPay 的表單頁。
- `bank_account`、`store_number`、`bank_id`、`pay_currency`、`mobile`、`id_no` 目前屬於補充欄位，不是每個渠道都會回吐。

### 回應欄位

| 欄位 | 說明 |
|---|---|
| `data.order_id` | 商戶訂單號 |
| `data.transaction_id` | 系統訂單號 |
| `data.view_url` | 前往付款頁的 URL |
| `data.user_name` | 付款人姓名 |
| `data.bill_price` | 訂單金額 |
| `data.real_price` | 實際付款金額，目前等同訂單金額 |
| `data.bank_no` | 第一個 `bank_account` 值，若有提供 |
| `data.bank_name` | `Bank Transfer` / `Convenience Store` / `channel_code` |
| `data.bank_from` | `bank_id` |
| `data.bank_owner` | `user_name` |
| `data.remark` | 由 `pay_currency`、`mobile` 組出的補充字串 |

### 目前不會回傳的歷史欄位

- `data.qr_url`
- `data.expired`
- `data.alipay_qrcode`
- `data.rate`

## 查單

### 請求欄位

| 欄位 | 型別 | 必填 | 說明 |
|---|---|---|---|
| `pay_customer_id` | string | Y | 商戶 ID |
| `pay_apply_date` | string | Y | Unix timestamp |
| `pay_order_id` | string / array[string] | Y | 商戶訂單號，可單筆或陣列 |
| `pay_md5_sign` | string | Y | MD5 簽名 |

### 回應說明

- `data` 為陣列。
- `pay_channel_id` 會依內部 `channel_code` 再映射回對外編碼。
- `view_url` 會一併帶回。
- `rc_feedback.rate` 與 `rc_feedback.display_price` 目前固定為 `null`。

### 狀態碼

| `status` | 說明 |
|---|---|
| `0` | 待付款 / 處理中 |
| `2` | 已付款 |
| `5` | 付款失敗 |

## 商戶 callback

- provider 通知入站後，服務會將整理過的 RY 相容 payload POST 到 `pay_notify_url`。
- 商戶端需回傳純文字 `OK`。

### callback 狀態

| `status` | 說明 |
|---|---|
| `30000` | 訂單成功 |
| `50000` | 訂單失敗 |
| `10000` | 訂單處理中 |

## 目前已確認未支援

- 已移除渠道不再受理。
- 不提供支付寶相關欄位。
- 不提供 QR code / 匯率 / 過期時間等歷史回傳格式。
