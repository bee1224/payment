# 核心資料庫結構與帳務流程

> 最後更新：2026-07-11

## 核心資料表

| 資料表 | 用途 |
|---|---|
| `merchants` | 下游商戶基本資料 |
| `merchant_api_keys` | 商戶 API key hash 與生命週期 |
| `merchant_balances` | 商戶餘額 |
| `orders` | 收款訂單 |
| `provider_transactions` | 收款 provider 交易紀錄 |
| `provider_callbacks` | provider callback 紀錄 |
| `payout_orders` | 代付訂單 |
| `payout_beneficiaries` | 代付收款人資訊 |
| `payout_transactions` | 代付送單 attempt |
| `payout_callbacks` | 代付 provider callback |
| `merchant_payout_callback_tasks` | 商戶 callback outbox / retry |
| `ledger_entries` | 帳務總帳 |

## 目前關鍵事實

- 代付已經有本地持久化，不再是純 HTTP 代理
- `api_key_hash` 與 `merchant_api_keys.key_hash` 現在都以 hash 形式保存
- merchant key 已支援 rotate / revoke / validate

## 上板前要注意

- 不要把正式商戶原始 API Key 直接寫進 Git 追蹤檔
- VPS `.env` 裡的 bootstrap `MERCHANT_API_KEY` 只用於初始化與本地驗證邏輯
