# 代收資料流

> 更新日期：2026-07-09
> 目前代收 provider 僅有 `NewebPay`，收款渠道已縮減為 7 種。

## 1. 建立訂單

```mermaid
flowchart LR
    A["商戶系統"] -->|"POST /api/pay_order
    pay_customer_id / pay_order_id / pay_amount
    pay_channel_id / pay_notify_url / pay_md5_sign"| B["payment-service"]
    B -->|"驗簽 / 驗時間 / pay_channel_id -> channel_code"| C["DepositService"]
    C -->|"寫入 orders / provider_transactions"| D[("MySQL")]
    C -->|"channel_code -> newebpay"| E["NewebPay gateway"]
    B -->|"回傳 order_id / transaction_id / view_url"| A
```

### 補充

- `pay_channel_id` 目前只接受 `1000`、`1001`、`1002`、`1005`、`1006`、`1007`、`1008`。
- 內部 `DepositService` 目前只允許 `CREDIT`、`APPLEPAY`、`GOOGLEPAY`、`WEBATM`、`VACC`、`CVS`、`BARCODE`。
- provider mapping 目前全部都指向 `newebpay`。

## 2. 前往付款頁

```mermaid
flowchart LR
    A["商戶前端"] -->|"開啟 view_url"| B["GET /api/v1/deposits/{order_no}/redirect"]
    B -->|"輸出 auto-submit HTML form"| C["payment-service"]
    C -->|"POST MerchantID / TradeInfo / TradeSha"| D["NewebPay MPG"]
```

### 補充

- `view_url` 是 payment-service 的 redirect 頁，不是 provider 原始 URL。
- 前端只需要導向 `view_url`，不需要自己組 NewebPay 表單。

## 3. Provider 通知與商戶 callback

```mermaid
flowchart LR
    A["NewebPay"] -->|"POST provider notification"| B["payment-service"]
    B -->|"驗證通知 / 更新 orders / provider_transactions"| C["DepositService"]
    C -->|"入帳 ledger / merchant_balances"| D["Ledger"]
    B -->|"POST RY 相容 callback 到 pay_notify_url"| E["商戶系統"]
    E -->|"回傳 OK"| B
```

### 補充

- callback 狀態目前只整理成 `30000`、`50000`、`10000` 三種。
- 商戶 callback 內容是 payment-service 統一後的 RY 相容 JSON，不是 NewebPay 原始格式。
- 商戶端必須回 `OK`。

## 4. 查單

```mermaid
flowchart LR
    A["商戶系統"] -->|"POST /api/query_transaction"| B["payment-service"]
    B -->|"依 merchant_code + merchant_order_no 查 orders"| C[("MySQL")]
    B -->|"回傳 status / transaction_id / pay_channel_id / view_url"| A
```

### 補充

- 查單回應會把內部 `channel_code` 轉回 `pay_channel_id`。
- `view_url` 會一起回傳，方便商戶重新導流到付款頁。

## 5. 涉及資料表

| 資料表 | 用途 |
|---|---|
| `orders` | 代收訂單主檔 |
| `provider_transactions` | provider request / response / notify 資料 |
| `provider_callbacks` | provider callback 紀錄 |
| `ledger_entries` | 入帳流水 |
| `merchant_balances` | 商戶餘額 |
| `merchants` | 商戶設定與 callback URL |
| `payment_providers` / `payment_channels` | provider 與渠道映射 |
