# 代收資料流

> 最後更新：2026-07-11

## 流程摘要

1. 商戶呼叫 `POST /api/pay_order`
2. 系統驗簽與驗欄位
3. 建立本地收款訂單
4. 回應 `view_url`
5. 使用者前往付款頁
6. provider 通知我方
7. 我方更新訂單與帳務
8. 我方 callback 商戶 `pay_notify_url`

## callback 語意

- 商戶收到的是我方整理後的 gateway 相容 JSON
- 不是 NewebPay 原始通知格式
- 商戶需回 `OK`

## 目前對外身份

- 商戶眼中：`RIG001 Gateway`
- 上游收款通道：NewebPay
