# 文件索引與閱讀順序

> 最後校對：2026-07-07

## 先看這段就夠的總結

- 收款已是正式可對外使用能力。
- 代付不再只是單純代理，現在已經有本地提款單、審核、送 RY、查單補查、回調入帳、商戶結果回調與取消機制。
- `/api/payments/*` 是 **RY provider contract 代理層**。
- `/api/payouts/*` 是 **我方正式提款工作流 API**。
- 若文件和程式互相矛盾，以「程式碼 + 測試 + 現行路由」為準。

## 文件權威順序

1. **對方契約**：[01_RY對方站台完整技術文件](../對外溝通用/01_RY對方站台完整技術文件.md)
2. **我方正式對外文件**：提供商戶 / 對方系統商閱讀
3. **我方內部規格文件**：描述目前實作狀態與限制
4. **草案 / 待辦 / 確認清單**：只能視為規劃與追蹤，不代表已上線

## 使用原則

1. 要對外發文件時，優先從 `對外溝通用` 取。
2. 要確認現在程式到底有沒有做到，優先看 `03_`、`04_`、`06_`。
3. `草案`、`確認清單`、`待辦` 不可直接當成對外承諾。
