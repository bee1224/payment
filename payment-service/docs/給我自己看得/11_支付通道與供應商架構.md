# 支付通道與供應商架構

> 文件定位：描述目前程式的 Provider 邊界；圖中的未註冊供應商不代表已完成串接。

## 架構

```mermaid
flowchart LR
  A[下游商戶] --> B[本系統 API]
  B --> C{通道選擇}
  C --> D[藍新代收：已實作]
  C -.未註冊.-> E[其他代收 Provider]
  B --> F[RY 代付 HTTP client：已實作]
  D --> G[收款訂單]
  E -.未實作.-> G
  G --> H[帳本]
```

## Provider 介面

```mermaid
flowchart TD
  A[CreatePayment] --> B[產生付款資料]
  C[VerifyNotification] --> D[驗簽與解密]
  D --> E[回傳統一通知格式]
```

- 代收使用 `DepositGateway`：目前只註冊 `newebpay`。
- 代付使用獨立的 RY `PayoutClient`：目前不是通用 `PayoutGateway` registry，也沒有本地訂單持久化。

## 正式環境

```mermaid
flowchart LR
  A[設定檔] --> B[正式金鑰]
  B --> C[Provider]
  C --> D[正式閘道]
```
