# 支付通道與供應商架構

> 最後更新：2026-07-11

## 分層理解

1. 下游商戶
2. 我方 `RIG001 Gateway`
3. 上游供應商 / provider
4. 實際支付通道

## 目前架構

- 收款：
  - 下游商戶 -> `RIG001 Gateway` 收款接口
  - `RIG001 Gateway` -> NewebPay
- 代付：
  - 下游商戶 -> `RIG001 Gateway` workflow API
  - `RIG001 Gateway` -> 上游 gateway provider

## 重要語意

- `RIG001 Gateway` 是我方正式對外角色
- `gateway` 是程式內部抽象命名
- `RY` 只剩歷史命名或相容格式，不再當正式品牌

## 目前已實作到的層級

- 收款：建單、查單、provider callback、商戶 callback
- 代付：建單、審核、送單、補查、provider callback、商戶 callback、retry、resend、merchant key lifecycle
