# Project Docs Read

## 工作流程

1. 先從任務中的關鍵字、元件名稱、路由、功能或作業類型，定位 `docs/` 中最直接相關的文件；先讀取一份，僅在資訊不足時再加入下一份。
2. 不要預設列舉或閱讀整個 `docs/`，也不要預設掃描整個 Repository。
3. 文件已足以取得完成當前任務所需資訊時，停止搜尋且不要讀取程式碼。
4. 文件不足時，逐步閱讀最相關的程式碼、設定、測試或 migration；每一步只擴大到能補足缺口的範圍。
5. 需要擴大搜尋範圍前，先在 commentary 說明缺少哪些資訊、為何現有文件或檔案不足，以及下一個要檢視的範圍。
6. 發現文件與程式碼不一致時，列出具體差異（文件位置、程式碼位置與影響），提醒使用者；不要自行假定哪一方正確。
7. 取得任務所需資訊後立即停止探索，轉入回答、分析或實作。

## 選擇原則

- 優先選擇標題、檔名或內容直接對應任務的文件，而非通用概覽。
- 對單一問題先讀取最可能回答它的一份文件；只有該文件未涵蓋必要細節時才讀取下一份。
- 程式碼探索應從文件所指向的元件開始；文件未指向時，以任務直接提及的檔案或功能為起點。
- 回報所讀取的文件與擴大範圍的原因，保持簡短。
